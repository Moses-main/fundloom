# Admin Operations Guide

## Objectives
- Track platform issues quickly.
- Resolve abuse, fraud, and service incidents safely.
- Maintain high-confidence auditability.

## Core Admin Feature Set
1. **Issue Intake**
   - Sources: user reports, automated detectors, system alerts.
   - Severity: P1/P2/P3 classification.
2. **Case Management**
   - Status: Open → Investigating → Actioned → Resolved.
   - Assignment and SLA timer.
3. **Resolution Actions**
   - Campaign: flag, pause, verify, remove.
   - User: warn, limit, suspend, reinstate.
4. **Audit Trail**
   - Every action logs actor, reason code, timestamp, evidence links.
5. **Metrics**
   - Time-to-triage, time-to-resolution, recurrence rates.

## Minimum Production SLAs
- P1: triage within 15 minutes.
- P2: triage within 2 hours.
- P3: triage within 1 business day.

## Post-incident Review
- Root cause summary
- Impacted entities
- Corrective and preventive actions
